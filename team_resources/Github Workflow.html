Github Workflow

Throughout our first group projects, it's apparent that Github workflow is crucially important. So here is the clearest picture I can conjure to explain how it is done (made using Sketch and Skitch).

In addition, hear are some important tips of things to avoid because they will lead to pain:

  • Multiple team members working in the same file
  • A team member not rebasing and going through the rebase process and creating a pull request
    • Code reviewers should catch this, it's part of the job of code review
    • Code review cannot happen if a team member (or, potentially, their pair) merges their own pull request
  • Team members do more than the scope of their feature in their feature's branch
  • When you change branches or rebase, close your text editor. Sometimes editors will try to save changes that you do not intend to save

Working in groups, may seem slow at first, but very few big projects were done by a single contributor.
